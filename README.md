# 0xkey Enclave KeyOps Skill

Agent skill / runbook package for operating 0xkey Nitro enclave key ceremonies.
This repository ships **four role skills**, one per ceremony seat. Each role
skill is fully self-contained so it can be installed independently and only
exposes the actions that role is actually allowed to take.

## Skills in this repository

| Skill name | Slash command (OpenClaw, sanitized) | Owns |
|------------|-------------------------------------|------|
| `0xkey-keyops-coordinator` | `/0xkey_keyops_coordinator` | Manifest generation / envelope, deploy render/apply, boot-genesis / boot-standard / post-share, K8s overlay, attestation, verify, key-forward |
| `0xkey-keyops-manifest` | `/0xkey_keyops_manifest` | Review bundle verify, `approve-manifest`, approvals bundle packaging |
| `0xkey-keyops-share` | `/0xkey_keyops_share` | Share-request bundle verify, `ceremony share-extract`, `proxy-re-encrypt-share`, wrapped-shares bundle |
| `0xkey-keyops-builder` | `/0xkey_keyops_builder` | qos_client / qOS release / pivot binaries / pivot hashes / ECR digests / builder-handoff |

Each role skill is `disable-model-invocation: true`, so Cursor / Codex /
Claude Code only load it when the user names it explicitly (or invokes the
matching OpenClaw slash command).

## Repository layout

```text
core/                            Single editable source. Edit files here.
  SECURITY.md
  PRINCIPLES.md
  WORKFLOWS.md                   Full ceremony narrative (Coordinator only)
  config.prod.example.json
  scripts/
    enclave_keyops.py            stdlib CLI wrapping qos_client + kubectl
    role_init.py                 stdlib role workspace initializer
  references/
    operator-prompts.md          per-role start prompts
    provisioning-matrix.md       what each role must produce / consume
    qos-client-platform.md       operator-client matrix & SHA256 policy
    workspace-rules.md           shared workspace + security baseline
    roles/
      coordinator.md
      manifest-set-member.md
      share-set-member.md
      builder.md

skills/                          Installable skill packages (fan-out of core/).
  0xkey-keyops-coordinator/      Auto-gen contents + hand-written SKILL.md
  0xkey-keyops-manifest/         Auto-gen contents + hand-written SKILL.md
  0xkey-keyops-share/            Auto-gen contents + hand-written SKILL.md
  0xkey-keyops-builder/          Auto-gen contents + hand-written SKILL.md

tools/
  sync-skills.py                 Regenerates `skills/<role>/` from `core/`.
                                 Run with `--check` in CI to detect drift.

tests/                           Stdlib unittest suite (no external deps).
```

`core/` is the only place to edit shared content. Per-role copies in
`skills/<role>/` are byte-identical to `core/` and are regenerated by
`tools/sync-skills.py` — never hand-edit a per-role copy.

## Install / use with agents

Pick the one role skill that matches your seat in the ceremony. Skill names
listed below are the canonical `name:` values from each `SKILL.md`.

### Per-agent skill paths

`<skill>` is one of `0xkey-keyops-coordinator`, `0xkey-keyops-manifest`,
`0xkey-keyops-share`, or `0xkey-keyops-builder`.

| Agent | User-level (per machine) | Project-level (per repo) |
|-------|--------------------------|--------------------------|
| Cursor | `~/.cursor/skills/<skill>` | `<repo>/.cursor/skills/<skill>` |
| Codex CLI | `~/.codex/skills/<skill>` | `<repo>/.codex/skills/<skill>` |
| Claude Code | `~/.claude/skills/<skill>` | `<repo>/.claude/skills/<skill>` |
| OpenClaw | `~/.openclaw/skills/<skill>` | `<repo>/.openclaw/skills/<skill>` |
| Antigravity | `~/.antigravity/skills/<skill>` | `<repo>/.antigravity/skills/<skill>` |
| GitHub Copilot Skills | `~/.copilot/skills/<skill>` | `<repo>/.github/copilot/skills/<skill>` |
| Generic AgentSkills (multi-platform shared) | `~/.agents/skills/<skill>` | `<repo>/.agents/skills/<skill>` |

### One-time symlink (recommended; tracks `git pull` automatically)

From inside the cloned repo:

```bash
REPO_ROOT="$PWD"

install_skill() {
  local skill_name="$1"
  local agent_dir="$2"
  mkdir -p "$agent_dir"
  ln -sfn "$REPO_ROOT/skills/$skill_name" "$agent_dir/$skill_name"
}

# Example: a Manifest Set member on Cursor (user-level)
install_skill 0xkey-keyops-manifest "$HOME/.cursor/skills"

# Example: a Coordinator on Codex + Claude Code (user-level)
install_skill 0xkey-keyops-coordinator "$HOME/.codex/skills"
install_skill 0xkey-keyops-coordinator "$HOME/.claude/skills"
```

`ln -s` keeps each installed skill in sync with `git pull` without copying.
Use `cp -R` only on platforms that reject symlinked skill directories.

OpenClaw's `skills.load.allowSymlinkTargets` defaults to off for workspace /
project-agent roots; symlinks under `~/.openclaw/skills/` and
`~/.agents/skills/` are allowed by default but the SKILL.md realpath must
stay inside its resolved skill directory.

### `npx skills add` (when the repo is publicly resolvable)

```bash
npx skills add <github-org>/<repo> --skill 0xkey-keyops-manifest \
  -a cursor -a codex -a claude-code -a openclaw
```

`<github-org>/<repo>` is the GitHub path to this repository; `--skill` names
one of the four role skills above. Skip this route if the repo is private and
fall back to the symlink section.

### ClawHub install (OpenClaw)

```bash
openclaw skills install <skill-slug>
```

`<skill-slug>` is the ClawHub-published slug for the desired role skill, which
matches the `name:` field (e.g. `0xkey-keyops-manifest`).

### Project-level install

Copy or symlink the skill into the project repo (for example
`<repo>/.cursor/skills/0xkey-keyops-manifest`) when you want a team to share
the same version via the project's git history. The agent's project-level
skills directory takes precedence over the user-level one, so a project-pinned
version cannot be silently shadowed by a stale user-level install.

### Agents without first-class skill support

Hand the agent the contents of `skills/<role>/` explicitly:

```text
SKILL.md
SECURITY.md
PRINCIPLES.md
config.prod.example.json
references/roles/<your-role>.md
references/operator-prompts.md
references/workspace-rules.md
references/qos-client-platform.md
references/provisioning-matrix.md
scripts/enclave_keyops.py
scripts/role_init.py
```

And prompt:

```text
Treat this directory as the 0xkey-keyops-<role> runbook package.
Start from SKILL.md and follow its action whitelist; do not execute another
role's workflow.
```

## Operator usage

Start by telling the agent your exact role. Each line below loads the matching
role skill directly; no role-router round-trip needed.

```text
我是 0xkey KeyOps 的 Builder / Release 角色。
请使用 0xkey-keyops-builder skill，只执行 Builder 角色流程。
```

```text
我是 0xkey KeyOps 的 Deployment Coordinator。
请使用 0xkey-keyops-coordinator skill，只执行 Coordinator 角色流程。
```

```text
我是 0xkey KeyOps 的 Manifest Set 成员，alias 是 manifester1。
请使用 0xkey-keyops-manifest skill，只执行 Manifest Set member 角色流程。
```

```text
我是 0xkey KeyOps 的 Share Set 成员，alias 是 share-member1。
请使用 0xkey-keyops-share skill，只执行 Share Set member 角色流程。
```

If a user opens a conversation without naming their role (or with a
cross-role utility action like "verify this bundle"), ask which seat they
have before invoking any role skill. Routing logic belongs in the
conversation, not in a separate skill.

For longer per-role start prompts, see
[`core/references/operator-prompts.md`](core/references/operator-prompts.md)
(also present inside each role skill).

## Safety rules

- Never commit or share `.secret`, `.share`, wrapped-share plaintext, DR
  secret, or extracted sensitive ceremony directories.
- Member long-term key material must live outside the role workdir, preferably
  in an operator-controlled encrypted key vault.
- Manifest / Share members do not need AWS, EKS, or kubectl access; their
  skills do not whitelist any such command.
- Dangerous steps require explicit human confirmation: manifest approval,
  share re-encryption, deploy apply, and post-share. `--yes` does not bypass
  these.

See `core/SECURITY.md` and role documents under `core/references/roles/` (or
the synced copies inside each role skill) before running a ceremony.

## Maintenance

```bash
# Regenerate per-role copies after editing anything under `core/`.
tools/sync-skills.py

# CI check: exit 2 if any role skill has drifted from `core/`.
tools/sync-skills.py --check

# Run the unit test suite (Python 3.11+).
python3 -m unittest discover -t . -s tests -v
```
